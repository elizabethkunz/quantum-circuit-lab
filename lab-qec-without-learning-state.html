How does the surface code catch errors without reading the data?

Work through these five tasks in order. The first three build intuition using a small distance-3 patch — you will explore the grid anatomy, understand what a stabilizer measures, and see how errors light up alarms. Task 04 is the full challenge on a distance-5 patch.

Task 02

A stabilizer is a parity alarm.

The highlighted stabilizer continuously measures the joint Z-parity of its four neighboring data qubits. It returns +1 if an even number of them have been hit by X errors, and −1 (a violation) if the count is odd.

Click the four highlighted data qubits to toggle X errors on and off. Watch the stabilizer outcome change.

Focus stabilizer
Highlighted data qubits are toggleable
Parity rule
An X error on a data qubit flips its contribution to the parity. An even total leaves the stabilizer happy (+1). An odd total trips the alarm (−1). This works for any combination of errors — the outcome only encodes the count modulo 2, never the actual qubit state.
Toggle errors on the highlighted data qubits and watch the stabilizer outcome update.
Task 03

Explore how location shapes the syndrome.

Click any data qubit to toggle an X error on it. All stabilizers update immediately. Try corner, edge, and interior qubits — notice how many alarms each location triggers. You can add multiple errors and see how patterns combine.

This is the key insight behind syndrome decoding: the pattern of violations tells a decoder where errors are, without ever measuring the encoded logical state.

Location rules (d=3)
Corner qubit → 1 stabilizer
Edge qubit → 2 stabilizers
Interior qubit (center) → 4 stabilizers
Interior qubits produce the largest syndrome signature; corner qubits the smallest. On a larger patch, decoding algorithms exploit these signatures to identify likely error locations.

Task 05

Why this is quantum error correction.

The surface code does not work by measuring the data qubits. Measuring them would collapse any superposition stored in the logical qubit and destroy the quantum information. Instead, stabilizer measurements are non-destructive.

The key property: every Z stabilizer commutes with the logical operators X_L and Z_L. Measuring a stabilizer projects the state onto a definite error subspace without revealing whether the logical qubit is in |0_L⟩, |1_L⟩, or any superposition of them.

The syndrome tells you the error, not the data. When you identified the error location in Task 04, you did it using only the stabilizer outcomes — the content of the logical qubit played no role. A decoder on real hardware works exactly the same way: classify violations, infer the most likely error, apply a classical correction.

Code distance d = 5. A minimum-weight decoder can correct any pattern of (d−1)/2 = 2 or fewer independent X errors. The more violations in Task 04 you needed to reason about simultaneously, the closer you were to that limit. Beyond 2 independent errors on a d = 5 patch, a naive decoder starts failing — which is why scaling to larger d is the central engineering challenge in fault-tolerant quantum computing.
